========================================================
  平面轨迹生成软件 — 文件夹结构说明
========================================================

【运行方法】
  python main.py      ← 启动软件
  python keygen_vendor.py  ← 供应商生成激活码用


--------------------------------------------------------
一、根目录文件
--------------------------------------------------------

main.py
  整个软件的主程序，也是唯一的入口文件。
  它干了以下几件事：
    1. 启动时检查授权（调用 function/license_manager.py）
    2. 创建主窗口（仿照师兄软件的布局：Ribbon工具栏 + 右侧参数面板 + 左侧预览图）
    3. 把 Ribbon 工具栏的按钮和右侧参数面板的切换逻辑连在一起
    4. 调用 function/planar_trajectory.py 里的算法生成轨迹点
  你平时改功能、加参数、改界面，基本都在这个文件里改。

keygen_vendor.py
  供应商专用的激活码生成工具。
  用户把自己的机器码发给你，你输入机器码和授权天数，
  它就给你生成对应的激活码，你再把激活码发给用户。
  用户拿到软件后不需要这个文件，这个只你自己留着。


--------------------------------------------------------
二、function/ 文件夹  ← 软件的"大脑"，核心逻辑都在这里
--------------------------------------------------------

function/planar_trajectory.py
  轨迹生成的数学算法，两个核心函数：
    generate_raster()  — 栅形轨迹，蛇形扫描线
    generate_spiral()  — 等弧长阿基米德螺旋线轨迹
  还有 save_trajectory_txt() 负责把轨迹点保存成 TXT 文件。
  输出格式：每个点 [X, Y, Z=0, Nx=0, Ny=0, Nz=1]
  如果以后要改轨迹算法、加新轨迹类型，就改这个文件。

function/license_manager.py
  软件授权/激活的全部逻辑：
    get_hardware_id()         — 读取本机 CPU+主板序列号，生成机器码
    generate_activation_code()— 根据机器码+授权天数生成激活码（HMAC加密）
    activate()                — 验证激活码，写入 license.dat
    verify_license()          — 每次启动时校验 license.dat 是否有效
  如果要改加密方式或授权逻辑，就改这个文件。

function/license.dat
  激活成功后自动生成的授权文件，里面存了：
    机器码、激活时间、到期时间、防篡改哈希值等
  这个文件不要手动改，改了授权校验就会失败报错。
  如果想重新激活，把这个文件删掉就行。

function/__init__.py
  空文件，作用是告诉 Python "这个文件夹是一个包"，
  这样 main.py 才能用 from function.xxx import xxx 来导入。
  不用动它。


--------------------------------------------------------
三、GUI/ 文件夹  ← 界面框架，直接从师兄软件复制过来的
--------------------------------------------------------

这整个文件夹是师兄软件的 Ribbon 工具栏框架，
我们的软件直接复用了它，没有任何修改。
它们之间的关系如下（从大到小）：

GUI/RibbonWidget.py
  最顶层的 Ribbon 工具栏容器，就是那个 Tab 栏整体（像 Office 的工具栏）。
  里面可以添加多个 Tab（选项卡），比如"轨迹规划""授权""退出"。

GUI/RibbonTab.py
  每一个选项卡（Tab）的容器，比如"轨迹规划"这一整个 Tab。
  里面可以添加多个 Pane（面板组），比如"栅形轨迹""螺旋线轨迹""输出"。

GUI/RibbonPane.py
  每一组按钮的面板，比如"栅形轨迹"这一组。
  下面有标题文字，里面可以放若干个按钮。
  两个 Pane 之间有一根竖线分隔（RibbonSeparator）。

GUI/RibbonButton.py
  Ribbon 里的大按钮（带图标+文字，竖排显示）。
  每个按钮绑定一个 QAction，点击后触发对应功能。

GUI/Icons.py
  图标管理器，启动时把 icons/ 文件夹里所有 .png 文件全部加载进来，
  之后通过 get_icon("图标名") 就能拿到对应图标。

GUI/StyleSheets.py
  CSS 样式管理器，把 stylesheets/ 文件夹里的 .css 文件读进来，
  之后通过 get_stylesheet("样式名") 取到对应样式字符串。

GUI/__init__.py
  里面只有一个函数 gui_scale()，用来读取屏幕 DPI，
  让按钮大小在高分辨率屏幕上自动缩放，防止按钮变得太小。

GUI/RibbonScrollarea.py
  当一个 Tab 里按钮太多放不下时，提供横向滚动能力。
  我们目前按钮不多，基本用不到，但框架里有所以也复制过来了。

GUI/RibbonTextbox.py
  Ribbon 里嵌入文本输入框用的，师兄软件里用过，
  我们目前没用到，不用管它。


--------------------------------------------------------
四、icons/ 文件夹  ← 所有图标图片
--------------------------------------------------------

存放软件界面上用到的所有图标 (.png / .ico)，
全部从师兄软件复制过来。
GUI/Icons.py 启动时会自动扫描这个文件夹里所有 .png 文件，
文件名去掉 .png 后缀就是图标的名字，比如：
  zhexian.png  →  get_icon("zhexian")  →  栅形轨迹按钮的图标
  luoxuan.png  →  get_icon("luoxuan")  →  螺旋线轨迹按钮的图标
  exit.png     →  get_icon("exit")     →  退出按钮的图标
  license.png  →  get_icon("license")  →  授权管理按钮的图标

如果你想换图标，直接把新图片放进这个文件夹，
文件名和代码里 get_icon() 里的名字对应上就行。


--------------------------------------------------------
五、stylesheets/ 文件夹  ← 界面的 CSS 样式
--------------------------------------------------------

控制软件界面颜色、字体、边框等视觉效果，全从师兄软件复制。

main.css
  全局背景色设置：background-color: #dfe9f5（那个淡蓝色背景）

ribbon.css
  Ribbon 工具栏的整体样式：Tab 的颜色、选中状态、悬停橙色高亮等

ribbonButton.css
  大按钮（RibbonButton）的样式：
  悬停时出现的蓝色渐变边框效果

ribbonSmallButton.css
  小按钮的样式，和大按钮类似但尺寸不同

ribbonPane.css
  每组按钮面板（RibbonPane）的样式：字体 9pt 微软雅黑等

如果你想改界面颜色，改 main.css 和 ribbon.css 就够了。


--------------------------------------------------------
六、__pycache__/ 文件夹（自动生成，不用管）
--------------------------------------------------------

Python 运行时自动生成的编译缓存，
把 .py 源代码编译成 .pyc 字节码存在这里，目的是让下次启动更快。
不用动它，删掉也没关系（下次运行会重新生成）。


========================================================
  总结：你平时需要关心的文件
========================================================

  改功能/算法  →  function/planar_trajectory.py
  改界面/参数  →  main.py
  改授权逻辑  →  function/license_manager.py
  换图标      →  往 icons/ 里放新图片
  改颜色风格  →  stylesheets/main.css 或 ribbon.css
  生成激活码  →  运行 keygen_vendor.py

  其他文件夹（GUI/ icons/ stylesheets/ __pycache__/）
  基本不需要动，它们是支撑界面运行的底层框架。
